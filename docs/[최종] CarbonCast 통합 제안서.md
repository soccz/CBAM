## Executive Summary

**문제:** 2026년 1월 EU CBAM 시행. 한국 수출기업의 87%가 배출량 산정 방법을 모른다(KIEP 조사). 모르면 EU 기본값(+30% 마크업)이 적용되어 **실제보다 수십% 과다한 탄소 비용**을 부담한다.

**솔루션:** CarbonCast는 3개 레이어로 이 문제를 해결한다.

| 레이어 | 역할 | 핵심 기술 |
|:-------|:-----|:----------|
| **AI Engine** | 배출량 추정 + 산업 충격 분석 + 규정 자동 가이드 | Conformal Prediction, I-O + GNN, LLM Agent |
| **Trust Layer** | 제품별 탄소 이력 추적 + 데이터 무결성 | 블록체인 DPP, Federated Learning |
| **Financial Layer** | 심사 연결 + 포트폴리오 분석 + ESG 금융 | READIT OCR, PCAF, 여신 시스템 연동 |

**왜 하나금융인가:** READIT OCR(연 8만건 수출서류), 무역금융 25회 연속 1위, PCAF 금융배출량 산정 체계, 융합기술원 73명, ESG 37조 실적 — **수출 데이터 + 재무 데이터 + AI 기술을 동시에 가진 유일한 포지션**.

**현재 상태:** 합성 데이터 628개 업체 기반 프로토타입 작동 중. 하나금융 데이터 연결 시 즉시 실전 전환 가능.

## Part 1. 문제

### 1-1. CBAM이 뭔가 — 30초 요약

EU가 2026년 1월부터 시행하는 **탄소 관세**. 유럽에 수입되는 철강·시멘트·알루미늄 등에 탄소 비용을 부과한다. 한국 기업이 EU에 수출할 때, EU 탄소 가격과 한국 탄소 가격의 차이만큼 추가 비용이 발생한다.

단, **한 번에 100%가 아니다.** 단계적으로 적용된다:

| 연도 | 적용률 | 의미 |
|:-----|------:|:-----|
| 2026 | 2.5% | 시작은 소액 |
| 2030 | **48.5%** | 본격 증가 |
| 2034 | **100%** | 전액 부담 |

*출처: EU Regulation 2023/956, Annex IV (무상할당 단계적 폐지 스케줄)*

### 1-2. 진짜 문제: "배출량을 모른다"

CBAM 비용 공식은 단순하다: `(EU 가격 − 한국 가격) × 적용률 × 배출량`

**비용 공식이 문제가 아니다. "배출량"이 문제다.**

| 상황 | EU가 적용하는 값 | 결과 |
|:-----|:----------------|:-----|
| 실측 데이터 제출 | 기업 실제 배출계수 (예: 1.85 tCO₂/t) | 정상 비용 |
| 데이터 없음 | **EU 기본값 + 30% 마크업** (예: 3.90 tCO₂/t) | **2배 이상 과다 부담** |

*출처: EU IR 2025/2621, 기본값 마크업 스케줄 (2026: +10%, 2027: +20%, 2028~: +30%)*

**구체적 손해:**

| 품목 | EU 기본값 (2028, +30%) | 한국 실제 평균 | 차이 × €75 | 1만톤 수출 시 연간 과다 부담 |
|:-----|-----:|-----:|-----:|-----:|
| 철강 반제품 | 3.90 | ~1.85 | €154/톤 | **약 22억 원** |
| 시멘트 클링커 | 1.13 | ~0.80 | €25/톤 | **약 3.6억 원** |
| 알루미늄 1차 | 1.90 | ~1.50 | €30/톤 | **약 4.4억 원** |

*출처: 기본값 — EU IR 2025/2621, S&P Global (2025.10); 한국 평균 — 환경부 인벤토리, 포스코 ESG 보고서*

### 1-3. 한국 기업의 현실

- 배출량 산정 방법을 **모르는 기업 87%** (KIEP 중소기업 CBAM 대응 조사)
- EU 수출 기업 중 **대응계획 없음 54.9%** (KIEP)
- K-ETS 대상은 816개 대형 사업장. **중소 수출기업 대부분은 비대상** → 데이터 인프라 자체가 없음

*출처: KIEP 「수출중소기업 CBAM 대응방안 연구」, KDI 동향조사*

### 1-4. 은행에게 왜 문제인가

| 리스크 | 구체적 | 금액 |
|:-------|:-------|:-----|
| PF 심사 부실 | 탄소비용 미반영 → IRR 과대 추정 → 부실 대출 | 금감원 1차 스트레스테스트: 무대응 시 **45.7조 손실** |
| 고객 이탈 | CBAM 대응 못 해주는 은행에서 이탈 | CBAM 대상 한국 수출 **51억 달러** |
| 규제 | 금감원 2차 기후 스트레스테스트(2026) — 산업별 탄소비용 노출도 데이터 요구 | 은행 자본비율 **최대 3.8%p 하락** |

*출처: 금감원 기후 스트레스테스트 1차 결과 (2025.3.18), 한국은행 보고서*

### 1-5. 시간표 — 왜 지금인가

| 시점 | 이벤트 |
|:-----|:-------|
| **2026.01** | CBAM 본격 시행 (적용률 2.5%) |
| 2026 | 금감원 2차 기후 스트레스테스트 |
| **2027.02** | EU DPP(디지털 제품 여권) 의무화 시작 (배터리) |

| 2030 | CBAM 적용률 48.5% — 비용 본격 증가 |
| 2034 | CBAM 적용률 100% — 전액 부담 |

*출처: EU Regulation 2023/956; ESPR(지속가능제품 에코디자인 규정) 2024.7 발효; EU 집행위 2025.12 다운스트림 입법안*

### AI Engine

#### A-1. 배출량 추정 — Conformal Prediction

**목적:** 실측 데이터 없는 기업의 배출계수를 **수학적 보장 구간으로 추정**하여, EU 기본값 대비 비용 절감 가능성을 사전 판단.

> **중요:** 이 추정치는 **EU CBAM 신고에 직접 사용하는 것이 아니다.** EU는 "실측 or 기본값"만 인정한다 (EU Regulation 2023/956). CarbonCast의 추정치는 **"실측 데이터 확보의 우선순위를 정하고, 비용 최적화 전략을 수립하는 의사결정 지원 도구"**로 사용한다.

**작동 방식:**

```text
입력: 업종(KSIC), 에너지 사용량, 전력 사용량, 연료 종류, 생산량
  ↓
[XGBoost/LightGBM] — K-ETS 명세서 데이터로 학습
  ↓
[MAPIE Conformal Prediction] — 모델 예측 위에 보정 구간 생성
  ↓
출력: "배출계수 1.85 tCO₂/t, 90% 보장 구간 [1.62, 2.08]"
  ↓
의사결정: "EU 기본값(3.90)보다 상한(2.08)도 낮음
          → 실측 데이터 확보하면 톤당 €136 절감 가능
          → 실측 투자 ROI: 1만톤 수출 시 연 20억 절감 vs 검증 비용 수천만 원"
```

**Conformal Prediction을 쓰는 이유:**
- 일반 quantile regression: "90% 구간"이라 해도 실제 커버리지가 85%일 수 있음
- Conformal prediction: **분포 가정 없이(distribution-free) 커버리지를 수학적으로 보장** (Angelopoulos & Bates, 2021, 인용 1000+)
- 이 차이가 금융에서 중요: "90%라고 했는데 실제론 85%"면 리스크 관리가 무너짐

**학습 데이터:**
- K-ETS 관리업체 명세서 (data.go.kr/15053947, REST API) — 업체별 배출량 + 에너지 소비량 + 업종
- 한국에너지공단 마이크로데이터 (data.go.kr/15075918, Open API) — 사업체별 에너지/CO₂
- **한계:** K-ETS는 816개 대형 사업장 대상. 중소기업 적용 시 selection bias 존재 → 이를 보정하기 위해 산업연관표 부문별 배출원단위를 prior로 활용

**프로토타입 결과 (합성 628개 업체):**
- Stage 1 (업종 + 매출만): R² = 0.776, MAPE 20.1%
- Stage 2 (에너지 데이터 포함): R² = 0.778, MAPE 8.0%
- 철강 고로 기준: EU 기본값 대비 **톤당 €156 절감**, 8개 기업 연간 합계 **697억 원 절감**

**이해충돌 해소:** "기업에겐 낮게, 은행엔 높게"가 아니라 **구간의 양쪽을 용도별로 사용**:
- 고객 서비스: 하한(1.62)에 맞춰 절감 전략 제안
- 리스크 관리: 상한(2.08)으로 보수적 추정 → 충당금/리스크 가중치 산정

#### A-2. 산업 충격 분석 — Leontief I-O + GNN

**목적:** CBAM 비용이 한국 산업 전체에 어떻게 퍼지는지 시각화.

**2단계 접근:**

| 단계 | 방법 | 용도 |
|:-----|:-----|:-----|
| **Baseline** | 레온티에프 역행렬 `(I−A)⁻¹` | 381개 부문 직접/간접 효과. 밀리초 내 계산. 투명하고 감사 가능 |
| **확장** | GNN (PyTorch Geometric) | **기업 수준 거래 네트워크** 확보 시 비선형 전파/피드백 루프 학습 |

> **왜 레온티에프 baseline을 먼저 두는가:** 381개 부문 수준 IO표에서 GNN이 레온티에프를 유의미하게 이기기 어렵다 (부문 수준은 선형 가정이 충분). GNN의 진짜 가치는 **기업 수준 거래 네트워크**(수만 노드)에서 발휘된다. 하나금융의 무역금융 거래 데이터가 연결되면 이 확장이 가능하다.

**학술 근거:** Stanford AAAI 2025 — "Learning Production Functions for Supply Chains with GNNs" (기존 대비 6~50% 성능 향상, 단 기업 수준 데이터 사용 시)

**데이터:**
- 한국은행 산업연관표 (380개 기본부문, 2020년 기준, ecos.bok.or.kr 무료)
- 확장 시: 하나금융 무역금융 거래 데이터 (기업 간 수출입 관계)

**킬러 장면:** 철강 노드 클릭 → 비용 충격이 자동차부품 → 완성차 → 기계로 퍼지는 네트워크 애니메이션. "CBAM은 철강만의 문제가 아닙니다."

#### A-3. CBAM 컴플라이언스 Agent — LLM + RAG

**목적:** CBAM 규정이 수백 페이지이고 계속 바뀐다. 기업별 맞춤 가이드를 자동 생성.

**작동:**

```text
[규정 변경 모니터링]
  EU 관보/TAXUD 사이트 → RAG 인덱싱 → 변경 감지 시 알림

[기업별 맞춤 가이드]
  기업 HS코드 + 수출국 + 배출량 → LLM Agent가 해당 기업에 적용되는 규정 추출
  → "귀사는 HS 7208.51(열연강판), CBAM 직접 대상.
     2026년 1월까지 필요 서류: CBAM 인증서, 탄소배출량 검증보고서.
     2028년부터 다운스트림 확대 시 추가 영향: [해당사항 없음/있음]"

[보고서 자동 생성]
  CarbonCast 배출량 추정 + 비용 시나리오 → CBAM 신고서 초안 자동 생성
```

**학술 근거:** ACM (2025) "A Multi-Agent RAG Framework for Regulatory Compliance"; 한국과학기술정보연구원 (2025) THE-RAG 기반 HS코드 자동 분류 시스템

**하나금융 연결:** 융합기술원 NLP셀 역량 + READIT OCR로 수출서류 자동 파싱 → Agent 입력

#### A-4. 탄소가격 시나리오 분석

**목적:** NGFS 시나리오 기반으로 2026~2034 CBAM 비용 경로를 산출.

> **표현 주의:** "가격 예측"이나 "인과 분석"이 아니라 **"시나리오 기반 민감도 분석"**. 관측 데이터만으로 EU ETS 가격의 인과관계를 주장하는 것은 학술적으로 과도하다 (World Bank Research Observer, 2023). CarbonCast는 NGFS가 제공하는 시나리오 경로를 그대로 사용하고, AI는 **단기(1주) 가격 범위 참고 정보**만 제공한다.

**프로토타입 결과 (실제 EU ETS 데이터 1,619일):**
- LightGBM Quantile: MAE 3.15 EUR, MAPE 4.4%, 방향성 정확도 66.1%
- 1등 변수: EU ETS 변동성, 2등: 천연가스 가격 — 기존 학술 연구와 일치
- **이 모델의 용도:** "현재 가격 움직임이 NGFS 어떤 시나리오에 가장 가까운가?"를 판단하는 보조 지표

### Trust Layer

#### T-1. Digital Product Passport (DPP) — 2027 EU 의무화 선제 대응

**왜 이게 블록체인이 진짜 필요한 지점인가:**

2028년 CBAM 다운스트림 확대 시, 완제품(자동차부품, 가전)의 **"내재 탄소(embedded carbon)"**를 제품 단위로 추적해야 한다. 원재료(철강) → 부품 → 완성품까지 탄소가 어떻게 흘러가는지.

이건 **단일 기업의 DB로 안 된다.** 공급망의 여러 기업이 참여하는 **다자간 데이터 공유**가 필요하고, 데이터 변조 방지가 필요하다. → 블록체인의 본래 용도.

**구조:**

```text
[철강사] → 배출량 데이터 → 블록체인 기록
    ↓
[부품사] → 철강 내재 탄소 + 자체 공정 탄소 → 블록체인 기록
    ↓
[완성품사] → 부품 내재 탄소 + 자체 공정 탄소 → 블록체인 기록
    ↓
[DPP QR코드] → 제품별 총 탄소 발자국 조회 가능
    ↓
[EU CBAM 신고] → DPP 데이터로 내재배출량 증빙
```

**선례:** Catena-X (BMW, Mercedes-Benz 참여) — 자동차 공급망 PCF(Product Carbon Footprint) 추적. 2026년 supplier-specific PCF 데이터 운영 중.

**하나금융의 역할:** 공급망 금융(SCF)을 제공하는 은행이 DPP 인프라의 **신뢰 앵커(trust anchor)** 역할. "하나금융이 검증한 탄소 데이터"라는 신뢰.

*출처: ESPR 2024.7 발효, EU DPP 레지스트리 2026.7 운영 시작, 배터리 여권 2027.2 의무화*

#### T-2. Federated Learning — 데이터 비공개 공동 모델

**문제:** 배출량 추정 모델을 고도화하려면 더 많은 기업 데이터가 필요. 하지만 기업들은 에너지 사용량, 공정 데이터를 공유하지 않는다.

**해결:** 연합학습(FL). 각 기업이 자기 데이터를 보유한 채 모델 파라미터만 공유.

```text
하나금융 (오케스트레이터) — 글로벌 모델 관리
     ↑ 모델 업데이트만 수집 (원천 데이터 X)
     │
  ┌──┼──┬──┬──┐
  기업A 기업B 기업C 기업D  — 각각 로컬 모델 학습
  (데이터는 (데이터는 ...
   기업 내부에 유지)
```

**하나금융의 포지션:** 3,500개 무역금융 고객 중 EU 수출 기업을 FL 노드로 참여시키면, **한국 최대 규모의 CBAM 배출량 추정 모델**이 만들어진다. 데이터는 각 기업에 남고, 모델만 하나금융에.

**학술 근거:** FedCarbonNet (Springer, 2025) — AI + Blockchain + IoT 결합 탄소 관리 프레임워크, 50개 노드 운영, MAE 0.082 Mt

**블록체인 연결:** FL 라운드별 모델 해시를 블록체인에 기록 → 모델 변조 방지 + 감사 추적

### Financial Layer

#### F-1. READIT OCR → CBAM 대상 자동 판별

**이미 있는 것:** READIT이 수출환어음 서류에서 HS코드, 수입국, 금액, 수량을 자동 추출 (연 8만건)

**CarbonCast가 추가하는 것:** 추출된 HS코드 → CBAM 대상 여부 자동 판정 → 2028년 다운스트림 확대 대상 여부까지

```text
[수출서류 업로드] → [READIT OCR] → HS코드 7208.51 추출
    ↓
[CBAM 매핑 DB] → "열연강판, CBAM 직접 대상 (2026~)"
    ↓
[CarbonCast] → 배출량 추정 → 비용 시나리오 → 심사 연결
```

*출처: 관세청 「EU CBAM 규제품목 한-EU 품목번호 연계표」 (2026.02 공개)*

#### F-2. 재무 영향 변환 → 여신 심사

CBAM 비용 → 차주 재무 영향 → 신용지표 변동:

```text
EBITDA_adj = EBITDA − CBAM 비용
이자보상배율_adj = EBITDA_adj / 이자비용
→ 신용등급 변동 추정 (예: BBB → BB+)
```

**데이터:** DART Open API (무료) — 상장사 재무제표 자동 수집. 비상장 중소기업은 여신 심사 시 제출한 재무 데이터 활용.

#### F-3. 포트폴리오 CBAM 노출도 → 스트레스테스트

PCAF 체계(7개 범주 금융배출량)에 CarbonCast 시나리오 엔진을 연결:

- 여신 포트폴리오 업종별 CBAM 비용 노출도
- 시나리오별(NGFS) 잠재 손실 추정
- **금감원 2차 기후 스트레스테스트에 직접 입력 가능한 포맷**

#### F-4. ESG 금융 상품 연계

"CBAM 비용을 줄이는 설비 투자"에 ESG 우대금리 대출:

- 하나 ESG 지속가능연계 대출 (금리 인하 최대 1.2%p, 최장 5년) 기존 상품과 연계
- CarbonCast가 "이 투자로 배출계수가 X → Y로 줄어서 CBAM 비용이 Z억 절감"을 정량화 → 대출 심사 근거

*출처: 한국금융신문 (2025.07) — 하나 ESG 지속가능연계 대출 공급 규모 149억원*

## 부록: 기술 상세

### Conformal Prediction 구현

```python
from mapie.regression import MapieRegressor
from lightgbm import LGBMRegressor

# 기존 모델
base_model = LGBMRegressor(n_estimators=500, max_depth=6)

# Conformal Prediction 래핑
mapie = MapieRegressor(base_model, method="plus", cv=5)
mapie.fit(X_train, y_train)

# 90% 보장 구간
y_pred, y_pis = mapie.predict(X_test, alpha=0.1)
# y_pis[:, 0] = 하한, y_pis[:, 1] = 상한
# 커버리지 ≥ 90% 수학적 보장 (distribution-free)
```

### CBAM 비용 공식

```text
CBAM 비용(원/년)
  = 수출량(톤) × 배출계수(tCO₂/t) × EU수출비중(%)
    × [EU ETS 가격(€) − K-ETS 가격(€)] × CBAM 적용률(%)
    × 환율(원/€)
```

### CBAM Phase-in 적용률 (EU Regulation 2023/956)

| 연도 | 2026 | 2027 | 2028 | 2029 | 2030 | 2031 | 2032 | 2033 | 2034 |
|:-----|-----:|-----:|-----:|-----:|-----:|-----:|-----:|-----:|-----:|
| 적용률 | 2.5% | 5% | 10% | 22.5% | 48.5% | 61% | 73.5% | 86% | 100% |
